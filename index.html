Platform Identity

Not a trading bot.
Not a price oracle.

SentinelAI is a structured intelligence platform that connects geopolitical events, macroeconomic signals, and market sentiment to produce risk-weighted analysis on any financial instrument.

The output is always a probability-weighted scenario ‚Äî never a point prediction. Always with a full reasoning trace from source to conclusion.

"What is the macro risk profile of Brent crude given current Middle East tensions and the Fed's rate posture?"
‚Äî Example query, answered in under 90 seconds

‚úÖ What SentinelAI Does

‚Üí Analyzes geopolitical events with long historical memory
‚Üí Detects short-term market sentiment shifts
‚Üí Applies GARCH + Monte Carlo volatility modeling
‚Üí Runs Chronos-2 chart-behavior forecasting on any asset
‚Üí Cross-validates all outputs via a Critic agent
‚Üí Delivers PDF report + interactive dashboard

‚ùå What SentinelAI Does Not Do

‚Üí Predict exact prices
‚Üí Give buy/sell recommendations
‚Üí Trade autonomously
‚Üí Work with a fixed, hardcoded asset list

System Architecture

How the System Is Built

FRONTEND

React 18 + Vite

User submits a natural-language query specifying any asset. Dashboard renders risk heatmaps, timeline, reasoning trace, and scenario cards. Recharts + D3.js for all visualizations. Tailwind CSS for layout.

ORCHESTRATION

LangGraph + FastAPI

LangGraph manages the stateful agent pipeline as a directed acyclic graph. FastAPI exposes the REST endpoints. Celery + Redis handle async task queuing so the UI is never blocked during long inference chains.

STORAGE

MongoDB + Qdrant

MongoDB Atlas stores session state, agent reports, and full reasoning chains as flexible JSON documents. Qdrant is the vector database for RAG ‚Äî storing embedded geopolitical and macro documents. Long memory = long Qdrant index.

RAG Memory Architecture

Memory is not
one-size-fits-all.

The Geopolitical Agent and the Sentiment Agent use Qdrant differently ‚Äî because the nature of information they process has completely different decay rates.

"Political history compounds. Sentiment expires."
‚Äî Core memory design principle

The longer a geopolitical context window, the more accurately an agent can spot pattern recurrence ‚Äî a sanction echoing a 2014 precedent, or a currency crisis matching a 1997 template. Conversely, a sentiment window longer than ~14 days introduces noise: a Reddit panic from three weeks ago has zero relevance to today's risk.

Qdrant Index Configuration

10+ years
Context window
128K tokens
Retention policy
Indefinite
Why: Geopolitical patterns repeat across decades. A war's economic impact in 2014 is directly relevant to modeling a 2026 parallel.
AGENT 03 ‚Äî SENTIMENT ENGINE
Index depth
7‚Äì14 days only
Context window
4K tokens
Retention policy
Rolling TTL
Why: A Reddit panic from 3 weeks ago is noise today. Old sentiment vectors in the index corrupt current scoring ‚Äî so they expire automatically.

Prediction Engine

Chronos-2:
The chart reads
the chart.

Amazon's Chronos-2 is a pre-trained universal time series foundation model. It has never seen gold, oil, or any specific asset. It was trained purely on the behavior of curves ‚Äî how sequences of numbers move, compress, reverse, and trend.

You feed it any price series. It returns a probability distribution of what comes next ‚Äî expressed as 100 simulated future paths, from which we extract confidence bands.

Security Architecture

Two-layer defense:
Prevention + Detection

Security isn't a feature ‚Äî it's infrastructure. Every attack surface the Red Team could exploit is hardened at Layer 1 with OWASP best practices, then monitored at Layer 2 with an autonomous defense agent.

üõ°Ô∏è
LAYER 1

OWASP Best Practices

üîë Secrets Management

Zero hardcoded keys. All API keys in .env files never committed to Git. .gitignore enforced from day 1. Production keys stored in AWS Secrets Manager with automatic 90-day rotation. Attack surface: Environment variable injection ‚Äî mitigated via read-only container mounts.

üö´ XSS & Injection Prevention

React escapes all user input by default. MongoDB uses parameterized queries ‚Äî no string concatenation in queries. FastAPI uses Pydantic schema validation on every endpoint. CSP headers: script-src 'self' blocks inline scripts. Red Team target: User-provided ticker symbols ‚Üí sanitized via regex whitelist [A-Z0-9\^=\-\.]{1,12}.

‚è± Rate Limiting

Frontend: Button disabled after submission until pipeline completes. API: FastAPI Limiter ‚Üí 10 requests/minute per IP, 50/hour per authenticated user. Database: MongoDB connection pool capped at 100 concurrent. Red Team exploit: Distributed requests from multiple IPs ‚Üí caught by Layer 2 pattern detection.

üîê Auth & RBAC

Auth0 OAuth2 tokens with 1-hour TTL. Roles: Analyst (read-only), Admin (write access to pipeline config). JWT signature verified on every FastAPI request. Refresh tokens stored HttpOnly, SameSite=Strict. Attack vector: Token replay ‚Üí mitigated via short TTL + IP binding.

üåê CORS & HTTPS

CORS whitelist: https://app.sentinelai.io only. No wildcard origins. TLS 1.3 enforced. HSTS headers with 1-year max-age. Certificate pinning on mobile clients. Red Team angle: MITM via compromised CA ‚Üí mitigated via cert transparency logs.

LAYER 2

Autonomous Defense Agent

ARCHITECTURE
A separate Gemini 2.5 Flash agent consumes structured logs from all layers: Nginx access logs, FastAPI request logs, Celery task logs, MongoDB slow query logs, Qdrant search logs, Redis pub/sub events. Logs streamed to the agent every 10 seconds as batched JSON. Agent runs pattern detection + anomaly scoring, then intervenes autonomously when thresholds breach.
Real Attack Scenarios ‚Üí Agent Response
üö® Scenario: Rate Limit Bypass via Distributed IPs
Log pattern: 8 different IPs each sending 9 requests/min (just under 10/min limit), but all querying the same ticker within 2-minute window.
Agent reasoning: "Coordinated behavior ‚Äî likely botnet. Total effective rate: 72 req/min on single asset."
Action: Temporary IP range ban (CIDR block /24) for 30 minutes. Alert sent to admin dashboard.
üö® Scenario: MongoDB Injection Attempt
Log pattern: Pydantic validation error on ticker field: {"$ne": null} detected in POST body.
Agent reasoning: "NoSQL operator injection. Schema validation blocked it, but user is probing."
Action: IP flagged for 24h monitoring. If repeated within 1 hour ‚Üí permanent ban + log sent to security team.
üö® Scenario: Excessive Qdrant Vector Queries
Log pattern: Single authenticated user triggering 50 pipeline runs in 10 minutes (each run = 3‚Äì5 Qdrant searches).
Agent reasoning: "Possible attempt to exhaust vector DB or extract full index via API abuse."
Action: User session revoked. JWT invalidated. User account soft-locked pending admin review. Reasoning trace saved: "Usage 10x above normal analyst behavior."
üö® Scenario: Exfiltration via PDF Download Spam
Log pattern: User downloads 200 PDFs in 5 minutes without viewing dashboard between downloads.
Agent reasoning: "Automated scraping. Human analysts view reports before downloading. Bot signature detected."
Action: CAPTCHA challenge injected on next request. If failed ‚Üí 10-minute IP cooldown. Logs flagged for manual review.
Why an agent vs traditional SIEM rules? Fixed thresholds break with usage growth. The agent adapts ‚Äî it learns normal vs abnormal from historical log patterns. When it intervenes, it logs its full reasoning chain so security teams can audit decisions and tune sensitivity.
